Deploy do consorcio-ops (Next.js + Firestore só no cliente)
===========================================================

Não é necessário Firebase Admin nem JSON de conta de serviço no Netlify.
O app usa o SDK web + autenticação anônima e fala com o Firestore direto do browser.

1) Firebase — projeto, Firestore e Auth
   - Crie um projeto em https://console.firebase.google.com
   - Ative **Firestore** (Native mode).
   - Authentication → Sign-in method → habilite **Anonymous**.
   - Publique as regras em `firebase/firestore.rules` (ex.: `firebase deploy --only firestore:rules`).
   - Coleções usadas:
       administradoras/{id}
       planos/{id}
       vendas/{id}

2) Variáveis de ambiente (Netlify / Vercel / .env.local)
   - **Não** use `FIREBASE_SERVICE_ACCOUNT_JSON`.
   - Defina todas as `NEXT_PUBLIC_FIREBASE_*` do app web (mesmo projeto do Firestore):
       NEXT_PUBLIC_FIREBASE_API_KEY
       NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN
       NEXT_PUBLIC_FIREBASE_PROJECT_ID
       NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET
       NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID
       NEXT_PUBLIC_FIREBASE_APP_ID
       NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID (opcional)

3) Netlify
   - `netlify.toml` na raiz usa `npm run build`.

4) Vercel
   - `vercel.json` usa `npm run build`.

5) Build local
   - Preencha `NEXT_PUBLIC_FIREBASE_*` no `.env.local`.
   - `npm run build`

6) Segurança
   - As regras de exemplo permitem leitura/escrita para qualquer `request.auth != null` (inclui anônimo).
   - Para ambiente restrito, substitua por regras com e-mail verificado, domínio, ou outro modelo de acesso.
